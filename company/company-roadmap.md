# Company Roadmap — Security, Investigations, and Forensics Practice

## North Star

Build a high-trust boutique security, investigations, and forensics practice.

The current entry point is crypto / blockchain security because it combines smart contract audits, exploit reproduction, transaction analysis, wallet tracing, and public on-chain evidence.

The long-term company may expand beyond crypto into broader cybersecurity and digital investigations, including web security, incident response, OSINT, fraud analysis, cloud security, and security automation.

## Strategic Positioning

This company should not start as a fake “full-service cybersecurity firm.”

The correct path is:

```text
Start narrow:
Crypto / blockchain security + investigations

Build proof:
Labs, reports, case studies, public writeups, transaction analysis

Earn trust:
Small paid reports, small reviews, testimonials, referrals

Expand carefully:
Other cybersecurity branches only after enough skill, process, and credibility
```

## Core Business Thesis

Clients do not pay for “I know Solidity.”

Clients pay for:

```text
- finding security issues before attackers do
- explaining what happened on-chain
- tracing suspicious transactions
- proving exploit paths with evidence
- reducing risk before launch
- producing clear reports under pressure
```

Therefore, every learning task must produce one of these:

```text
- technical skill
- portfolio proof
- reusable methodology
- client-facing explanation ability
- service asset
- outreach asset
```

## Founder Role

The first role is not “CEO only.”

The first role is:

```text
founder-operator
```

Meaning:

```text
- learn enough technical depth to judge quality
- produce beginner-to-intermediate technical work
- build portfolio proof
- understand client problems
- create report templates and service processes
- sell small services honestly
- know when work is above current level
```

A pure CEO model is not realistic at the beginning because security companies require technical trust.

## Required Founder Competence

Minimum competence needed before selling serious work:

```text
1. Read small smart contracts.
2. Identify common vulnerability patterns.
3. Reproduce simple exploits.
4. Read block explorer transactions.
5. Understand external transactions vs internal calls.
6. Understand calldata, value, storage, logs, and events.
7. Write simple findings.
8. Build transaction timelines.
9. Explain risks in plain English.
10. Avoid overclaiming skill level.
```

## Ethics and Scope

This practice must stay ethical and defensive.

Allowed:

```text
- controlled lab exploitation
- defensive smart contract reviews
- transaction analysis
- wallet tracing from public data
- incident reconstruction
- security education
- responsible disclosure
```

Not allowed:

```text
- stealing funds
- unauthorized exploitation
- helping attackers hide funds
- phishing
- malware
- evasion
- bypassing access controls
- laundering guidance
```

## Operating Principle
